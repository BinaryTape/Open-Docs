# A2Aプロトコル

このページでは、KoogエージェントフレームワークにおけるA2A（Agent-to-Agent）プロトコルの実装について概要を説明します。

## A2Aプロトコルとは？

A2A（Agent-to-Agent）プロトコルは、AIエージェントが相互に、またクライアントアプリケーションと対話できるようにするための標準化された通信プロトコルです。
これは、一貫性があり相互運用可能なエージェント間通信を可能にする一連のメソッド、メッセージ形式、および動作を定義します。
A2Aプロトコルの詳細および詳細な仕様については、公式の[A2A Protocolウェブサイト](https://a2a-protocol.org/latest/)を参照してください。

## はじめに

**重要**: A2Aの依存関係は、`koog-agents`メタ依存関係にデフォルトでは含まれて**いません**。 
プロジェクトに必要なA2Aモジュールを明示的に追加する必要があります。

プロジェクトでA2Aを使用するには、ユースケースに基づいて依存関係を追加してください。

- **A2Aクライアントの場合**: [A2Aクライアントのドキュメント](a2a-client.md#dependencies)を参照してください。
- **A2Aサーバーの場合**: [A2Aサーバーのドキュメント](a2a-server.md#dependencies)を参照してください。
- **Koog統合の場合**: [A2A Koog統合のドキュメント](a2a-koog-integration.md#dependencies)を参照してください。

## 主要なA2Aコンポーネント

Koogは、クライアントとサーバーの両方に対してA2Aプロトコル v0.3.0の完全な実装を提供し、Koogエージェントフレームワークとの統合も提供しています。

- [A2Aサーバー](a2a-server.md)は、A2Aプロトコルを実装したエンドポイントを公開するエージェントまたはエージェントシステムです。クライアントからリクエストを受け取り、タスクを処理し、結果またはステータスの更新を返します。これはKoogエージェントとは独立して使用することもできます。
- [A2Aクライアント](a2a-client.md)は、A2Aプロトコルを使用してA2Aサーバーとの通信を開始するクライアントアプリケーションまたはエージェントです。これもKoogエージェントとは独立して使用できます。
- [A2A Koog統合](a2a-koog-integration.md)は、A2AとKoogエージェントの統合を簡素化するための一連のクラスとユーティリティです。これには、Koogフレームワーク内でのシームレスなA2Aエージェントの接続と通信のためのコンポーネント（A2A機能とノード）が含まれています。

さらなる例については、[examples](https://github.com/JetBrains/koog/tree/develop/examples/simple-examples/src/main/kotlin/ai/koog/agents/example/a2a)を確認してください。
