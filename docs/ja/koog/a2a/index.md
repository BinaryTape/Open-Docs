# A2Aプロトコル

--8<-- "versioning-snippets.md:beta"

このページでは、KoogエージェントフレームワークにおけるA2A (Agent-to-Agent) プロトコルの実装の概要について説明します。

## A2Aプロトコルとは？

A2A (Agent-to-Agent) プロトコルは、AIエージェント同士、またはAIエージェントとクライアントアプリケーションとの対話を可能にする標準化された通信プロトコルです。
一貫性があり相互運用可能なエージェント間通信を可能にするため、一連のメソッド、メッセージフォーマット、および動作を定義しています。
A2Aプロトコルの詳細情報および仕様については、公式の [A2A Protocol ウェブサイト](https://a2a-protocol.org/latest/) を参照してください。

## はじめに

**重要**: A2Aの依存関係は、`koog-agents` メタ依存関係にはデフォルトで**含まれていません**。
必要なA2Aモジュールをプロジェクトに明示的に追加する必要があります。

プロジェクトでA2Aを使用するには、ユースケースに応じて依存関係を追加してください。

- **A2Aクライアントの場合**: [A2Aクライアントのドキュメント](a2a-client.md#dependencies) を参照してください
- **A2Aサーバーの場合**: [A2Aサーバーのドキュメント](a2a-server.md#dependencies) を参照してください
- **Koog統合の場合**: [A2A Koog統合のドキュメント](a2a-koog-integration.md#dependencies) を参照してください

## A2Aの主要コンポーネント

Koogは、クライアントとサーバーの両方に対してA2Aプロトコル v0.3.0 の完全な実装を提供し、Koogエージェントフレームワークとの統合もサポートしています。

- [A2Aサーバー](a2a-server.md) は、A2Aプロトコルを実装したエンドポイントを公開するエージェントまたはエージェントシステムです。クライアントからのリクエストを受信し、タスクを処理して、結果やステータスの更新を返します。Koogエージェントとは独立して使用することも可能です。
- [A2Aクライアント](a2a-client.md) は、A2Aプロトコルを使用してA2Aサーバーとの通信を開始するクライアントアプリケーションまたはエージェントです。Koogエージェントとは独立して使用することも可能です。
- [A2A Koog統合](a2a-koog-integration.md) は、A2AとKoogエージェントの統合を簡素化する一連のクラスとユーティリティです。Koogフレームワーク内でのシームレスなA2Aエージェント間の接続と通信のためのコンポーネント（A2A機能およびノード）が含まれています。

その他の例については、[サンプル](https://github.com/JetBrains/koog/tree/develop/examples/simple-examples/src/main/kotlin/ai/koog/agents/example/a2a) を参照してください。
