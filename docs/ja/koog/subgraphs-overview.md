# 概要

このページでは、Koog フレームワークにおけるサブグラフの詳細な情報を提供します。これらの概念を理解することは、複数の処理ステップにわたってコンテキストを維持する複雑なエージェントのワークフローを作成するために不可欠です。

## はじめに

サブグラフは Koog フレームワークにおける基本的な概念であり、複雑なエージェントのワークフローを管理可能な連続したステップに分割できるようにします。各サブグラフは処理のフェーズを表し、独自のコンテキスト、責任、およびオプションのツールセットを持ちます。

サブグラフは、エージェントのワークフロー全体を表すグラフであるストラテジーの不可欠な構成要素です。ストラテジーの詳細については、[Custom strategy graphs](custom-strategy-graphs.md) を参照してください。

## サブグラフについて

サブグラフは、エージェントストラテジー内における自己完結した処理単位です。各サブグラフには以下の特徴があります。

- 一意の名前を持つ
- エッジで接続されたノードまたはサブグラフのグラフを含む
- ツールレジストリにある任意のツール、またはツールのサブセットを使用できる
- 前のサブグラフ（または最初のユーザー入力）から入力を受け取る
- 次のサブグラフ（または最終的な出力）に渡される出力を生成する

グラフ内でサブグラフのシーケンスを定義するには、エッジ接続を使用するか、`then` キーワードを使用してシーケンスを定義します。詳細については、[Custom strategy graphs](custom-strategy-graphs.md) を参照してください。

### サブグラフのコンテキスト

各サブグラフは、以下へのアクセスを提供するコンテキスト内で実行されます。

- 環境
- エージェントの入力
- エージェントの設定
- LLM コンテキスト（会話履歴を含む）
- ステートマネージャー
- ストレージ
- セッションとストラテジー

コンテキストはサブグラフ内の各ノードに渡され、ノードが操作を実行するために必要なリソースを提供します。
