Kotlin Multiplatformを使用すると、さまざまなプラットフォーム向けのアプリケーションを作成し、それらの間でコードを共有できるため、ユーザーがお使いのデバイスでアプリケーションを利用できるようになります。コード共有のためのコア技術であるKotlin Multiplatformと、Compose Multiplatform UIフレームワークによるサポート状況に応じて、各プラットフォームの安定性レベルは異なる場合があります。

このページでは、プロジェクトのニーズに適したプラットフォームを特定するのに役立つ情報と、それぞれの安定性レベルの詳細を提供します。

## Kotlinの一般的な安定性レベル

Kotlinにおける安定性レベルとその意味についてのクイックガイドです。

**試験的 (Experimental)** は、「トイプロジェクト（試作的なプロジェクト）でのみ試用してください」という意味です。

* 私たちはアイデアを試している段階であり、ユーザーに実際に触れてもらい、フィードバックを得たいと考えています。うまくいかない場合は、いつでも廃止される可能性があります。

**アルファ (Alpha)** は、「自己責任で使用してください。移行に伴う問題が発生することを想定してください」という意味です。

* 私たちはこのアイデアを製品化する意図がありますが、まだ最終的な形には達していません。

**ベータ (Beta)** は、「使用可能です。移行に伴う問題を最小限に抑えるよう最善を尽くします」という意味です。

* ほぼ完成しており、現在は特にユーザーからのフィードバックが重要です。
* ただし、100%完成しているわけではないため、（ユーザーからのフィードバックに基づくものを含め）変更の可能性があります。
* 最良のアップデート体験を得るために、事前に出される非推奨（deprecation）の警告に注意してください。

私たちは、「試験的 (Experimental)」、「アルファ (Alpha)」、「ベータ (Beta)」を総称して、**プレスタティック (pre-stable)** レベルと呼びます。

**安定 (Stable)** は、「最も保守的なシナリオであっても使用してください」という意味です。

* 完成しています。私たちは、厳格な[後方互換性ルール](https://kotlinfoundation.org/language-committee-guidelines/)に従って進化させていきます。

### コアとなるKotlin Multiplatformテクノロジーの現在のプラットフォーム安定性レベル

コアとなるKotlin Multiplatformテクノロジーの現在のプラットフォーム安定性レベルは以下の通りです。

| プラットフォーム | 安定性レベル |
|--------------------------|-----------------|
| Android                  | 安定 (Stable)   |
| iOS                      | 安定 (Stable)   |
| デスクトップ (JVM)       | 安定 (Stable)   |
| サーバーサイド (JVM)     | 安定 (Stable)   |
| Kotlin/WasmベースのWeb   | ベータ (Beta)   |
| Kotlin/JSベースのWeb     | 安定 (Stable)   |
| watchOS                  | ベータ (Beta)   |
| tvOS                     | ベータ (Beta)   |

* Kotlin Multiplatformは、ここに記載されている以上のネイティブプラットフォームをサポートしています。それぞれのサポートレベルを理解するには、[Kotlin/Nativeのターゲットサポート](https://kotlinlang.org/docs/native-target-support.html)を参照してください。
* Kotlin MultiplatformなどのKotlinコンポーネントの安定性レベルに関する詳細は、[Kotlinコンポーネントの現在の安定性](https://kotlinlang.org/docs/components-stability.html#current-stability-of-kotlin-components)を参照してください。

## Compose Multiplatform UIフレームワークの安定性レベル

Compose Multiplatform UIフレームワークのプラットフォーム安定性レベルとその意味についてのクイックガイドです。

**試験的 (Experimental)** は、「開発中」という意味です。

* 一部の機能がまだ利用できなかったり、提供されている機能にパフォーマンスの問題やバグがあったりする可能性があります。
* 将来的に変更される可能性があり、破壊的変更が頻繁に発生する可能性があります。

**アルファ (Alpha)** は、「自己責任で使用してください。移行に伴う問題が発生することを想定してください」という意味です。

* プラットフォームのサポートを製品化することを決定しましたが、まだ最終的な形には達していません。

**ベータ (Beta)** は、「使用可能です。移行に伴う問題を最小限に抑えるよう最善を尽くします」という意味です。

* ほぼ完成しているため、現在はユーザーからのフィードバックが特に重要です。
* まだ100%完成しているわけではないため、（ユーザーからのフィードバックに基づくものを含め）変更の可能性があります。

私たちは、「試験的 (Experimental)」、「アルファ (Alpha)」、「ベータ (Beta)」を総称して、**プレスタティック (pre-stable)** レベルと呼びます。

**安定 (Stable)** は、「最も保守的なシナリオであっても使用してください」という意味です。

* フレームワークは包括的なAPIサーフェスを提供しており、フレームワーク自体にパフォーマンスやその他の問題が発生することなく、美しく製品レベルで利用可能なアプリケーションを作成できます。
* APIの破壊的変更は、公式な非推奨の発表から2バージョン経過した後にのみ行うことができます。

### Compose Multiplatform UIフレームワークの現在のプラットフォーム安定性レベル

| プラットフォーム | 安定性レベル |
|--------------------------|-----------------|
| Android                  | 安定 (Stable)   |
| iOS                      | 安定 (Stable)   |
| デスクトップ (JVM)       | 安定 (Stable)   |
| Kotlin/WasmベースのWeb   | ベータ (Beta)   |
