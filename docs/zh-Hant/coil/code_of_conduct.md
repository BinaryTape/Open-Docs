# 行為準則

## 我們的承諾

為了營造開放且友好的環境，作為貢獻者與維護者，我們承諾為每位參與本專案及社群的人員提供一個無騷擾的體驗，無論其年齡、身材、身心障礙、族裔、性徵、性別認同與表達、經驗程度、教育程度、社會經濟地位、國籍、外貌、種族、宗教或性取向。

## 我們的標準

有助於營造積極正面環境的行為範例包括：

* 使用友好且具包容性的語言
* 尊重不同的觀點與經驗
* 體面地接受建設性的批評
* 專注於對社群最有利的事情
* 對其他社群成員展現同理心

參與者不可接受的行為範例包括：

* 使用性暗示語言或圖像，以及不文明的性關注或挑逗
* 惡意挑釁 (Trolling)、侮辱／貶低性的評論，以及個人或政治攻擊
* 公開或私下的騷擾
* 在未經明確許可的情況下，發布他人的私人資訊，例如實體地址或電子郵件地址
* 其他在專業場合中可被合理視為不當的行為

## 我們的責任

專案維護者負責釐清可接受行為的標準，並應針對任何不可接受的行為採取適當且公正的糾正措施。

專案維護者有權力且有責任移除、編輯或拒絕與本行為準則不符的評論、提交 (commit)、程式碼、Wiki 編輯、問題 (issue) 及其他貢獻；對於其認為不當、具威脅性、攻擊性或有害的其他行為，維護者可對貢獻者進行暫時或永久封鎖。

## 適用範圍

本行為準則適用於所有專案空間，也適用於個人在公共場合代表專案或其社群時。代表專案或社群的範例包括：使用官方專案電子郵件地址、透過官方社群媒體帳號發文，或在線上或線下活動中擔任指定的代表。專案維護者可對專案的代表性進行進一步的定義與釐清。

## 執行

若發現虐待、騷擾或其他不可接受的行為，可透過 colin at colinwhite.me 聯絡專案團隊進行舉報。所有投訴都將經過審查與調查，並根據具體情況做出必要且適當的回應。專案團隊有義務對事件的舉報者身分保密。具體執行政策的更多細節可能會另行發布。

若專案維護者未能誠信遵循或執行本行為準則，可能會面臨由專案領導層其他成員決定的暫時或永久性後果。
